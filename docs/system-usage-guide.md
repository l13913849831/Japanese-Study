# 日语学习系统使用说明

## 1. 文档目的

这份文档只说明当前仓库已经落地的能力、运行方式和主使用路径。

当前代码状态以 `2026-04-29` 为准，已经不是最早的“词库准备工具”形态，而是一个带本地账号的日语学习 MVP，包含两条学习线和一条统一主路径：

- 单词学习：词库、学习计划、今日卡片、复习记录、学习工作台
- 知识点学习：知识点 CRUD、Markdown 导入预览、知识点复习、知识点仪表盘
- 账号与偏好：注册、登录、登出、当前用户、账户资料、默认学习顺序
- 主路径与收尾：统一工作台、完成反馈、薄弱项、复盘导出、近 7 天复盘

仍然不包含：

- Keycloak
- 微信登录
- 多端同步
- `.apkg` 完整包导出

## 2. 仓库结构

- `backend/`: Spring Boot 3 + Java 21 + PostgreSQL + Flyway
- `frontend/`: React 18 + TypeScript + Vite + Ant Design
- `docs/`: 使用说明、接口文档、路线文档
- `.trellis/`: 工作流、规范、任务记录

## 3. 当前模块

后端模块：

- `user`: 注册、登录、账户资料、用户设置
- `wordset`: 词库、词条、导入预览与导入
- `studyplan`: 学习计划与生命周期动作
- `card`: 今日卡片、日历查询、复习提交、复习历史
- `dashboard`: 单词学习聚合视图
- `note`: 知识点、Markdown 导入预览、FSRS 复习、知识点仪表盘
- `weakitem`: 易错词、易错知识点、弱项摘要
- `template`: Anki / Markdown 模板列表、创建、更新、预览
- `backup`: 整体备份导出、恢复准备、安全快照、确认恢复
- `exportjob`: 导出任务创建、列表、下载
- `shared`: 统一响应、异常、配置、通用支持

前端页面：

- `/login`
- `/account`
- `/dashboard`
- `/word-sets`
- `/study-plans`
- `/cards`
- `/notes`
- `/notes/review`
- `/notes/dashboard`
- `/weak-items`
- `/templates`
- `/backups`
- `/export-jobs`

## 4. 运行前准备

后端运行条件：

- JDK 21
- Maven 3.9+
- PostgreSQL

前端运行条件：

- Node.js 22+
- npm 10+

数据库环境变量：

- `JP_DB_URL`
- `JP_DB_USERNAME`
- `JP_DB_PASSWORD`
- `APP_EXPORT_BASE_DIR`

默认配置见 `backend/src/main/resources/application.yml`。

## 5. 启动方式

### 5.1 启动后端

```bash
cd backend
mvn spring-boot:run
```

后端默认地址：

```text
http://localhost:8080
```

启动时会：

- 连接 PostgreSQL
- 执行 Flyway `V1` 到 `V11`
- 初始化词库、计划、卡片、模板、导出、知识点相关表
- 暴露 `/api` 接口

健康检查：

```text
GET http://localhost:8080/api/health
```

### 5.2 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端默认地址：

```text
http://localhost:5173
```

Vite 会把 `/api` 代理到后端 `http://localhost:8080`。

## 6. 当前已经落地的核心能力

### 6.1 单词学习线

- 创建词库
- 浏览词库列表
- 词条列表分页与关键词 / 等级 / 标签过滤
- 单词条创建、更新、删除
- CSV / `.apkg` 导入预览
- 导入确认后正式写入词条
- 创建、查看、更新学习计划
- 学习计划 `activate / pause / archive`
- 查询某计划某天的今日卡片
- 查询学习日历
- 为卡片提交 `AGAIN / HARD / GOOD / EASY` 评分
- 查看卡片复习历史
- 查看单词学习仪表盘

### 6.2 知识点学习线

- 手动创建、编辑、删除知识点
- Markdown 上传后先做拆分预览
- 预览阶段可编辑标题、内容、标签并删除草稿项
- 支持公共标签预填
- 导入后生成独立知识点，不保留源文件概念
- 查询当日知识点复习队列
- 按 FSRS 规则提交知识点评分
- 查看知识点复习历史
- 查看知识点仪表盘

### 6.3 模板、导出与备份

- 查看 Anki 模板
- 查看 Markdown 模板
- 创建与更新两类模板
- 对模板做草稿预览
- 创建账号整体备份并直接下载本地 zip
- 上传本地备份包做恢复准备
- 下载安全快照并确认覆盖恢复
- 创建导出任务
- 查看导出任务列表
- 下载导出结果

### 6.4 主路径、闭环与复盘

这部分对应现在的 `P1-A / P1-B / P1-C`：

- `/dashboard` 不只是统计页，而是今日主路径入口
- `/cards` 和 `/notes/review` 都有连续会话、层级提示和完成后的衔接动作
- `/weak-items` 负责薄弱项补强和收尾解释
- `/export-jobs` 负责把阶段结果导出成复盘材料，同时保留历史记录
- `/dashboard` 还提供近 7 天复盘、高峰日和风险信号

### 6.5 账号与访问

* 本地注册
* 本地登录
* 登出
* 查看当前用户
* 修改显示名
* 修改密码
* 修改默认学习顺序
* 当前使用 session cookie 做 Web 登录态

### 6.6 学习入口

- `/` 会跳到 `/dashboard`
- `/login` 是本地账号入口
- `/account` 是账户资料和默认学习顺序入口
- `/dashboard` 是统一今日学习工作台
- 工作台前端组合了单词学习仪表盘、知识点仪表盘、薄弱项和近 7 天复盘
- 工作台可直接跳到 `/cards`、`/weak-items`、`/notes`、`/notes/review`、`/notes/dashboard`、`/export-jobs`
- 从工作台进入复习时，会把当前选中的日期带进目标会话；单词线还会一并带上当前计划上下文

## 7. 页面与主用途

### 7.1 `/login`

- 本地注册和登录
- 当前是本地账号 + session cookie
- 登录失败时直接显示错误信息
- 注册后会自动建立会话

### 7.2 `/account`

- 修改显示名
- 切换默认学习顺序
- 修改密码

### 7.3 `/dashboard`

- 今日总览
- 当前推荐动作
- 单词学习摘要
- 知识点学习摘要
- 近 7 天复盘
- 风险信号和薄弱项
- 选定日期后，快捷入口会按该日期直接打开对应会话

### 7.4 `/word-sets`

- 管理词库
- 维护词条
- 上传 CSV / `.apkg`
- 先预览再确认导入

### 7.5 `/study-plans`

- 创建和编辑学习计划
- 选择模板
- 切换计划生命周期状态
- `reviewOffsets` 只保留旧计划兼容，不再驱动单词长期复习

### 7.6 `/cards`

- 优先直接打开当前计划的当天会话
- 如果当前计划不可用，则自动回退到第一个激活计划
- 只有没有当前计划且没有激活计划时，才提示先去创建或激活计划
- 进入连续单词复习会话
- 主区聚焦当前卡片与评分动作
- 计划/日期切换、手动跳题、最近历史降到辅助侧栏或下方辅助区
- 后端用 FSRS 动态生成下一张长期复习卡
- 当天回队尾和薄弱轮仍由前端会话态处理
- 会话完成后，会提示回工作台、切到知识点线、去薄弱项或导出复盘材料

### 7.7 `/notes`

- 管理知识点
- 上传 Markdown
- 调整导入预览草稿
- 确认导入

### 7.8 `/notes/review`

- 进入连续知识点复习会话
- 先看标题主动回忆
- 按需显示内容
- 主区只保留当前知识点与评分动作
- 日期切换、手动跳题、最近历史放在辅助区
- 评分后自动继续下一条
- 会话完成后，会提示回工作台、切到单词线、去薄弱项或导出复盘材料

### 7.9 `/notes/dashboard`

- 查看知识点总量、到期量、掌握度分布、最近趋势、最近新增

### 7.10 `/weak-items`

- 查看易错词和易错知识点
- 手动把项目移出弱项列表
- 了解为什么会进入薄弱轮
- 跳回工作台或导出复盘材料

### 7.11 `/templates`

- 创建、编辑、预览 Anki / Markdown 模板

### 7.12 `/backups`

- 创建当前账号整体备份并直接下载
- 上传本地备份包，启动恢复准备
- 先下载恢复前安全快照，再确认覆盖恢复
- 免费版不保留服务端托管备份历史

### 7.13 `/export-jobs`

- 作为学习收尾入口时，导出复盘材料
- 作为管理入口时，查看历史导出任务
- 下载文件

## 8. 推荐使用顺序

### 8.1 先登录和设偏好

1. 在 `/login` 注册或登录
2. 到 `/account` 设显示名和默认学习顺序
3. 回到 `/dashboard` 看今天的推荐动作

### 8.2 单词主路径

1. 在 `/word-sets` 创建词库并导入词条
2. 在 `/study-plans` 创建学习计划并激活
3. 在 `/dashboard` 看单词线今日任务和下一步动作
4. 直接从 `/dashboard` 进入 `/cards` 做连续复习
5. 做完后根据提示去 `/notes/review`、`/weak-items` 或 `/export-jobs`

### 8.3 知识点主路径

1. 在 `/notes` 手动创建知识点，或上传 Markdown 做预览导入
2. 在 `/notes/dashboard` 查看当日积压和趋势
3. 直接从 `/dashboard` 或 `/notes/dashboard` 进入 `/notes/review` 做连续复习
4. 做完后根据提示去 `/cards`、`/weak-items` 或 `/export-jobs`

## 9. 当前限制

- 仍然是本地账号体系，不含 Keycloak / 微信登录 / 多端同步
- 当前还没有管理员后台、角色权限模型、JWT/API token
- 免费版整体备份当前只支持“生成后直接下载 / 从本地上传恢复”，不保留服务端长期托管历史
- 工作台是前端组合现有接口，不是新的后端聚合接口
- 单词线已切到 FSRS 运行时调度，但 `reviewOffsets` 仍作为兼容字段保留在计划接口里
- 导出页的复盘入口已经有了，但历史管理、模板规则和更复杂的导出编排仍然偏轻
- 长周期统计能力还比较薄，但已经有近 7 天复盘和风险信号
