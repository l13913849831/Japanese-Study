# P0-b-3: implement restore flow with safety snapshot

## Goal

实现免费用户的两阶段恢复后端链路：上传本地备份包后，系统先生成并提供 `SAFETY_SNAPSHOT` 下载，再在用户确认后执行覆盖恢复。

## What I already know

* `P0-b-1` 已经定义了备份包格式和 manifest。
* `P0-b-2` 已经实现了 `MANUAL_BACKUP` 导出下载。
* 免费用户不保留服务端长期备份历史。
* 免费用户恢复前，必须先下载一份 `SAFETY_SNAPSHOT` 再继续恢复。

## Requirements

* 需要新增恢复准备接口，接收本地备份 `zip` 上传。
* 需要验证备份包格式和第一版必需文件。
* 需要在恢复准备阶段生成 `SAFETY_SNAPSHOT`。
* 需要新增安全快照下载接口。
* 需要新增恢复确认接口，在确认前禁止真正覆盖数据。
* 需要覆盖当前账号学习资产，但不修改登录凭据和系统默认资料。

## Acceptance Criteria

* [ ] 后端存在“准备恢复 / 下载安全快照 / 确认恢复”三段接口。
* [ ] 上传非法或缺文件的备份包会返回统一业务错误。
* [ ] 确认恢复前必须先下载安全快照。
* [ ] 恢复成功后，当前账号学习资产会被备份内容替换。
* [ ] 恢复流程不保留免费用户长期服务端托管备份。
* [ ] 有测试覆盖恢复准备、下载前置约束和恢复应用核心行为。

## Out of Scope

* 这轮不做前端页面
* 这轮不做 S3 / VIP 托管
* 这轮不做自动过期清理任务
* 这轮不做跨账号迁移

## Technical Notes

* 两阶段恢复需要一个短期 restore session 概念。
* 免费版可以用进程内临时会话保存解析后的备份与安全快照字节，不做长期持久化。
* 覆盖恢复需要按依赖顺序先清理再重建用户私有学习资产。
