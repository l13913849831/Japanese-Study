# brainstorm: unify learning main path

## Goal

把当前单词线和知识点线已经具备的学习能力，收敛成一个更明确的每日主路径：用户打开系统后，能更自然地知道先做什么、做完一条后去哪、当天任务是否已经完成。

## What I already know

* 当前统一入口已经是 `/dashboard`。
* `/cards` 和 `/notes/review` 已经都具备连续会话体验。
* 单词线和知识点线都已经有：
  * 今日待学习项
  * 连续会话
  * 弱项 / 薄弱项补救
  * 基础 dashboard / workbench 入口
* 当前仍主要依赖用户自己判断：
  * 先做单词还是先做知识点
  * 做完一条后是否继续另一条
  * 今天是否已经算“完成”
* 统一主路径会明显依赖“当前用户今天已进行到哪一步”的状态判断，而不是纯静态页面排序。

## Requirements (evolving)

* 需要明确“统一学习主路径”到底覆盖哪些页面和状态。
* 需要让用户在 `/dashboard` 看到更明确的下一步动作。
* 需要定义单词线与知识点线之间的衔接规则。
* 需要明确“今日完成”的判断方式和页面表达。
* 需要明确主路径规则是：
  * 固定顺序
  * 根据当日进度动态决定
  * 还是允许用户先设置默认偏好

## Acceptance Criteria (evolving)

* [x] 能明确当前主路径缺口，不和已完成的连续会话任务重复。
* [x] 能给出从 `/dashboard` 进入到当日学习结束的一条推荐路径。
* [x] 能说明单词线与知识点线如何衔接。
* [x] 能沉淀成下一步可实现的任务范围。

## Current First Slice

第一版先收成下面这条可交付主路径：

* 登录后默认进入 `/dashboard`
* `/dashboard` 根据用户偏好和当天剩余任务给出当前推荐动作
* 单词会话完成后，如果知识点还有待办，提示切到 `/notes/review`
* 知识点会话完成后，如果单词还有待办，提示回工作台继续单词线
* 两条主线都清空后，显示“今日主路径已完成”
* 若仍有薄弱项，作为可选加练入口展示，但不阻塞“今日完成”

这意味着第一版的“今日完成”定义为：

* 单词主线无当日待办
* 知识点主线无当日待办

薄弱项属于补强层，不算第一版的强制完成条件。

## Out of Scope

* 不扩展到 JLPT 课程、听力、口语等重模块
* 不先做登录、同步、多端

## Merged From: Learning Mode / Management Mode Split

原 `split learning mode and management mode after login` 不再作为独立任务保留，合并到 `P1-A`。

合并后的职责边界：

* 登录后默认进入学习模式，而不是直接暴露所有管理入口。
* 学习模式优先展示 `/dashboard`、`/cards`、`/notes/review` 这类每日动作。
* 词库、计划、模板、导入、导出历史等管理能力仍保留，但下沉到次级入口。
* 这不是权限模型；它是信息架构和默认路径问题。
* 真实身份、用户设置、登录态由 `P0-a` 提供，`P1-A` 只消费这些结果。

## Initial Breakdown

### 1. 入口层

* `/dashboard` 不只展示统计，还要表达“现在最该做什么”。
* 需要明确：
  * 默认推荐先做哪条线
  * 何时展示“继续单词会话”
  * 何时展示“继续知识点会话”
  * 是否允许用户设置“默认先单词 / 默认先知识点”的个人偏好

### 2. 会话衔接层

* 用户做完一段单词会话后，系统要不要直接建议转到知识点会话。
* 用户做完知识点会话后，系统要不要直接回工作台或进入收尾状态。
* 需要避免“每条线都做得不错，但整个产品没有一条日常主路径”的问题。
* 这里的判断需要依赖：
  * 当前用户已完成哪条线
  * 当前用户还有哪些当日待办
  * 是否存在尚未完成的薄弱轮

### 3. 完成判定层

* 需要定义“今天完成”到底是什么意思：
  * 单词主队列完成？
  * 单词 + 薄弱轮完成？
  * 知识点会话也完成？
  * 还是按用户当日目标完成？

## Emerging Direction

当前更稳的第一版方向不是直接做复杂“智能推荐”，而是：

1. 先允许用户设置一个轻量默认偏好：
   * 默认先单词
   * 默认先知识点
2. 工作台再结合当日实际进度做动态衔接：
   * 如果偏好线还没完成，优先继续该线
   * 如果偏好线已完成，自动切到另一条线
   * 如果两条线都完成，显示今日完成

这样可以避免第一版就引入过重的推荐规则，同时保留后续再演进到“按积压量动态决定”的空间。

## Dependency Note

* `P1-A` 的推荐规则最终应建立在 `P0-a` 提供的用户级偏好之上。
* “默认先单词 / 默认先知识点” 不再只视为本地临时偏好，而是正式用户设置的一部分。
* 因此 `P1-A` 的正式实现顺序应晚于 `P0-a`，避免后续再把路径规则从本地状态迁回用户设置。

### 4. 结果表达层

* 当用户完成当前推荐动作后，系统需要明确表达：
  * 已完成什么
  * 还有什么待做
  * 下一步推荐去哪
