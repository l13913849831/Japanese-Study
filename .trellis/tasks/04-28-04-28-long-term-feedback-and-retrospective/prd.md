# brainstorm: long-term feedback and retrospective

## Goal

让当前系统不只支持“今天学什么”，还支持“最近学得怎么样、哪里在积压、哪里长期偏弱”的长期反馈与复盘能力。

## What I already know

* 当前已有：
  * 今日工作台
  * 单词线 dashboard
  * 知识点线 dashboard
  * 弱项 / 薄弱项列表
* 当前较弱的是：
  * 周 / 月维度的反馈
  * 长期趋势总结
  * 按标签 / 类型 / 来源的复盘能力

## Requirements (evolving)

* 需要明确长期反馈最小可行范围。
* 需要区分：
  * 当日状态
  * 近期趋势
  * 长期弱点
* 需要说明哪些复盘能力依赖更完整的内容元数据。

## Acceptance Criteria (evolving)

* [ ] 能明确当前长期反馈的缺口。
* [ ] 能提出适合当前产品范围的复盘项，而不是重产品化扩张。
* [ ] 能收敛出下一轮值得先做的反馈能力。

## Out of Scope

* 不扩展到重课程模块
* 不先做复杂画像系统
